0. Lambda表达式

lambda表达式是C++11引入的”新”技术，之所以加引号是因为实际上其他语言如C#、Python中已经有了，只是C++在11版本中才刚刚推出而已。

1. 声明Lambda表达式

先上实际代码：

#include 

int main ()
{
    int a =1;
    int b =2;

    auto func = [=,&b](int c)->int{return b+=a+c;};

    return 0;
}

​ 简单的来说上面这段代码定义了两个变量a,b并分别赋值为1,2 。然后定义了一个auto类型的func变量，赋值为右侧的

[=,&b](int c)->int{return b+=a+c;}

​ 这段代码也就是一个基本的lambda表达式，它以[]作为表达式的开始，{}作为表达式的结束。[]中的=，&b表示，通过引用的方式传入b变量，其余变量通过值传递的方法传入。c是传入的形参（一个整型变量）， ->int 表示当前返回值因为Int类型，也就是说前面的auto在当前表达式里 可以替换成int， return b+=a+c;这是执行语句，也就是函数体。

Lambda表达式的完整声明如下：

[capture list] (params list) mutable exception-> return type { function body }

其中：

  • capture list：捕获外部变量列表
  • params list：形参列表
  • mutable指示符：用来说用是否可以修改捕获的变量
  • exception：异常设定
  • return type：返回类型
  • function body：函数体

除此完整版本之外，我们还可以省略某些成分来声明不完整的Lambda表达式，常见的如下：

序号 格式
1 [capture list] (params list) -> return type {function body}
2 [capture list] (params list) {function body}
3 [capture list] {function body}

  • 1省略了mutable类型的表达式，这种类型的表达式不能修改捕获列表中的值。

  • 2省略了返回值类型，但编译器可以根据以下规则推断出Lambda表达式的返回类型：

    （1）：如果function body中存在return语句，则该Lambda表达式的返回类型由return语句的返回类型确定；

    （2）：如果function body中没有return语句，则返回值为void类型。

  • 3中省略了参数列表，类似普通函数中的无参函数。

2.捕获外部变量

　2.1 捕获变量的方法

Lambda表达式可以使用其可见范围内的外部变量，如1中所说，lambda表达式起始部分[]内可以定义变量捕获的方式，从语法上来看，“[]”包括起来的部分叫做捕捉列表，捕捉列表由多个捕捉项组成，并以逗号分隔。捕捉列表有以下几种形式：

1.[var]表示值传递方式捕捉变量var；
2.[=]表示值传递方式捕捉所有父作用域的变量（包括this）；
3.[&var]表示引用传递捕捉变量var；
4.[&]表示引用传递方式捕捉所有父作用域的变量（包括this）；
5.[this]表示值传递方式捕捉当前的this指针。

上面提到了一个父作用域，也就是包含Lambda函数的语句块，说通俗点就是包含Lambda的“{}”代码块。上面的捕捉列表还可以进行组合，例如：

  • [=,&a,&b]表示以引用传递的方式捕捉变量a和b，以值传递方式捕捉其它所有变量;
  • [&,a,this]表示以值传递的方式捕捉变量a和this，引用传递方式捕捉其它所有变量。

不过值得注意的是，捕捉列表不允许变量重复传递，这会导致编译时期的错误。例如：

  • [=,a]这里已经以值传递方式捕捉了所有变量，但是重复捕捉a了，会报错的;
  • [&,&this]这里&已经以引用传递方式捕捉了所有变量，再捕捉this也是一种重复。

下面用一个简单的例子来说明：

#include 
int main()
{
    int a = 1;
    int b = 2;

    auto f = [&a,b]()
    {
        a++;
        b++;
        std::cout << a << "," << b << std::endl;
    };
    f();
    std::cin.get();
    return 0;
}

这里在实际开发环境中，b++语句报错的原因是变量b的捕获方式为值传递，这里没有传递一块内存地址给到lambda表达式里，而是取出了b的值2传入，所以这里如果想要正确执行b++并输出的话应该改成：

#include 
int main()
{
    int a = 1;
    int b = 2;

    auto f = [&a,b]()
    {
        a++;
        auto c = b;
        c++;
        std::cout << a << "," << c << std::endl;
    };
    f();
    std::cout << a << "," << b << std::endl;

    std::cin.get();
    return 0;
}

现在我们给代码打上断点并F5执行，如图可知lambda语句中a传入的是引用,而b则是一个整型变量，

F10继续执行可以看到此时变量a的值仍然为1，b的值为2，而传入lambda表达式中的a引用所指向的内存地址中所存的仍然是01 00 00 00，

F10继续执行可以看到，此时a的值随着a++;语句由1变成了2，

继续执行，成功将c的类型设置为int,值设置为b变量的值2，

继续执行，命令行窗口输出为

2,3
2,2

第一行输出是lambda表达式中传入a的引用再执行a++后a为2,将b的值传入并赋值给c再执行c++后c为3所以输出2,3

而第2行则是因为由于lambda表达式中改变了a的值，但并未改变b的值所以输出2,2

如下图所示，c变量在离开lambda表达式函数域后被销毁。

　2.2 操作值传递的变量

上述就是一个简单的lambda表达式的[]符号内参数的例子。

但是，如果我非要在lambda表达式中修改值传递变量的值呢？

也是可以做到的，只要在lambda表达式中加上一个mutable关键字就可以 了，如下所示：

注：mutable关键字在后面补充讲到它主要的用处。

#include 
int main()
{
    int a = 1;
    int b = 2;

    auto f = [&a,b]() mutable
    {
        a++;
        b++;
    };
    f();

    std::cin.get();
    return 0;
}

同样，打上断点，F5，此时a++语句已经执行结束，a==b==2,

F10,b的值成功完成自加操作被修改成了3，

3. lambda的使用

3.1 个人理解：匿名函数对象

我个人理解可以把lambda当做匿名函数对象来使用，如下代码,将要对IntArray数组所有元素执行的一个相同函数操作sum += x;作为参数传入std::for_each中：

#include
#include 
#include 

int main()
{

    int sum = 0;
    std::vector IntArray = { 1,2,3,4,5 };
    std::for_each(begin(IntArray), end(IntArray), [&sum](int x) {sum += x; });

    std::cin.get();
}

关于for_each中使lambda表达式和直接用for循环操作他们的区别以及为什么要使用for_each知乎上有一位大神举了个发苹果的例子，非常生动形象。

3.2 让代码更简洁

有了Lambda表达式，更多的方便了我们去写代码。比如以下代码：

#include

typedef enum
{
    add = 0,
    sub,
    mul,
    divi
}ECalculateType;

class CCalculate
{
public:
    CCalculate(int x, int y) :m_x(x), m_y(y) {}

    int operator()(ECalculateType i)
    {
        switch (i)
        {
        case add:
            return m_x + m_y;
        case sub:
            return m_x - m_y;
        case mul:
            return m_x * m_y;
        case divi:
            return m_x / m_y;
        }
    }

private:
    int m_x;
    int m_y;
};

int main()
{
    CCalculate addObj(10, 20);
    std::cout << addObj(add) << std::endl;
    std::cin.get();
    return 0;
}

​ 在有了lambda之后，我们可以将上述代码改写为：

#include

typedef enum
{
    add = 0,
    sub,
    mul,
    divi
}ECalculateType;


int main()
{
    int a = 10;
    int b = 20;

    auto func = [=](ECalculateType i)->int {
        switch (i)
        {
        case add:
            return a + b;
        case sub:
            return a - b;
        case mul:
            return a * b;
        case divi:
            return a / b;
        }
    };

    std::cout << func(add) << std::endl;
    std::cin.get();
    return 0;
}

4. mutable

​ mutable关键字的作用是将一个变量设置为可修改的，它主要用来解决有些const带来的问题。

​ const意思是“这个函数不修改对象内部状态”。

为了保证这一点，编译器也会主动替你检查，确保你没有修改对象成员变量——否则内部状态就变了。

​ mutable意思是“这个成员变量不算对象内部状态”。

如下代码所示，定义了一个CCountDebug的类，该类提供了一个getName的方法用来返回私有成员m_DebuggerName的值，一个m_DebugCount用来统计某个对象的访问次数（比如供debug用）。它变成什么显然并不影响对象功用，但编译器并不知道：它仍然会阻止一个声明为const的函数修改这个变量。

#include 
#include 

class CCountDebug
{
public:
    const std::string& getName() const
    {
        m_DebugCount++; //报错  正在通过常量对象访问m_DebugCount，所以无法对其进行修改
        return m_DebuggerName;
    }

private:
    std::string m_DebuggerName;
    int m_DebugCount = 0;

};

int main()
{

    CCountDebug CountDebug;
    CountDebug.getName();

    std::cin.get();
}

    const std::string& getName() const

这句代码中的第二个const表面getName方法是通过常量对象的形式进行访问的，并不能修改m_DebugCount的值。
要解决这个问题，只需要将m_DebugCount的声明改为 mutable int m_DebugCount = 0;即可，把这个计数变量声明为mutable，编译器就明白了：这个变量不算对象内部状态，修改它并不影响const语义，所以就不需要禁止const函数修改它了。
