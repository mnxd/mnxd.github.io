3.实现

下面的c实现经过了定制，以获得最佳性能。 代码中有两个分支:一个有效地剔除相加的三角形（#ifdef TEST_CULL），另一个在双面三角形上执行交集测试（#else）所有计算都被延迟，直到知道它们是必需的。例如，直到发现u的值在允许范围内，才计算v的值。

单面相交例程消除行列式（det）值为负值的所有三角形，这允许例程唯一的除法操作被延迟，直到确定相交为止。 对于阴影测试射线来说，这种划分是完全不需要的，因为我们所需要的只是三角形是否相交，

双面相交例程被强制执行除法操作以求出u和v值，或者，可以重写其函数以基于det的符号有条件地将u和v与0相比较。

该守则的某些方面值得特别注意。 边向量的计算可以作为一个前处理来完成，用边1和边2来代替vert1和vert2来进行其它计算，并且当顶点位置数据在三角形之间不共享时，不需要用边1和边2来代替vert1和vert2。

为了保证数值稳定性，消除平行射线的测试必须将其与ZERO周围的一个小间隔进行比较。 在适当调整EPSILON值的情况下，该算法是非常稳定的，如果只测试正面三角形，行列式可以与EPSILON进行比较，而不是与0进行比较（负行列式表示bach正面三角形）

u的值与三角形的一条边(u=0)比较，也与平行于该边但通过三角形的相对点的一条线(u=1)比较。 虽然实际上没有测试三角形的边，但是该第二个测试有效地排除了许多交点，而无需进一步的计算

#include 

#define EPSILON 0.000001
#define CROSS(dest,v1,v2) \
          dest[0]=v1[1]*v2[2]-v1[2]*v2[1]; \
          dest[1]=v1[2]*v2[0]-v1[0]*v2[2]; \
          dest[2]=v1[0]*v2[1]-v1[1]*v2[0];
#define DOT(v1,v2) (v1[0]*v2[0]+v1[1]*v2[1]+v1[2]*v2[2])
#define SUB(dest,v1,v2) \
          dest[0]=v1[0]-v2[0]; \
          dest[1]=v1[1]-v2[1]; \
          dest[2]=v1[2]-v2[2]; 

/*原始jgt代码*/
int intersect_triangle(double orig[3], double dir[3],
               double vert0[3], double vert1[3], double vert2[3],
               double *t, double *u, double *v)
{
   double edge1[3], edge2[3], tvec[3], pvec[3], qvec[3];
   double det,inv_det;

   /*查找共享vert0的两条边的向量*/
   SUB(edge1, vert1, vert0);
   SUB(edge2, vert2, vert0);

  /*开始计算行列式-也用于计算U参数*/
   CROSS(pvec, dir, edge2);

   /*若行列式接近零，则射线位于三角形平面内*/
   det = DOT(edge1, pvec);

   if (det > -EPSILON && det < EPSILON)
     return 0;
   inv_det = 1.0 / det;

  /*计算从vert0到光线原点的距离*/
   SUB(tvec, orig, vert0);

    /*计算U参数和测试界限*/
   *u = DOT(tvec, pvec) * inv_det;
   if (*u < 0.0 || *u > 1.0)
     return 0;

   /*准备测试V参数*/
   CROSS(qvec, tvec, edge1);

    /*计算V参数和测试界限*/
   *v = DOT(dir, qvec) * inv_det;
   if (*v < 0.0 || *u + *v > 1.0)
     return 0;

   /*计算t，射线与三角形相交*/
   *t = DOT(edge2, qvec) * inv_det;

   return 1;
}
