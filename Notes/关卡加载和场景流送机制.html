# UWorld::UpdateLevelStreaming

场景加载主要入口，这里主要是遍历 StreamingLevelsToConsider，通过 FStreamingLevelPrivateAccessor::UpdateStreamingState，调用对应 StreamingLevel 的

ULevelStreaming::UpdateStreamingState，根据 StreamingLevel 的状态机，处理它的加 / 卸载、可见性。

这里的 StreamingLevelsToConsider 数组有很多入口可以维护，最主要的是以下两个：

1.levelStreamingVolume

2.WorldComposition （没开启 WorldComposition 的情况下，StreamingLevelsToConsider 数组初值为空）

这两个也是 UE4 主要的场景流送机制

![[source/Img/Pasted image 20240111205354.png]]

# ULevelStreaming ::UpdateStreamingState

void ULevelStreaming::UpdateStreamingState (bool& bOutUpdateAgain, bool& bOutRedetermineTarget) 里面通过 CurrentState 来判断当前关卡状态从而执行不同的 add 或者 remove 操作

![[source/Img/Pasted image 20240111205612.png]]
在执行这个 switch case 判断之前要先判断当前 streaming 状态，是否有未完成 GC 的其他关卡，这其中的 LevelStreaming:: RequestLevel 就是关卡加载的入口代码，这里面会传入 PersistantWorld 和期望的加载方式，这里会用上面提到的 Package 加载逻辑，发起 Level Package 的加载，并注册加载完成回调 ——ULevelStreaming::AsyncLevelLoadComplete 。
![[source/Img/Pasted image 20240111205623.png]]

# ULevelStreaming ::AsyncLevelLoadComplete

这个是 Requestlevel 加载完成的回调函数。关键是 ULevelStreaming :: SetLoadedLevel。

其中主要是根据加载出来的 Level 的可见性，设置 CurrentState 为可见 ECurrentState:: LoadedVisible 或者不可见 ECurrentState:: LoadedNotVisible，并标记更新 UWorld:: UpdateStreamingLevelShouldBeConsidered，以便下一帧更新状态。UpdateStreamingLevelShouldBeConsidered 里其实就是往 StreamingLevelsToConsider 数组中添加 streaminglevel

# WorldComposition

WorldComposition 则是用于开放大世界的流送机制。它的工作原理是自动根据玩家位置加载周围一定范围内的场景，无需在场景中摆放 Volume 去触发。配置方法见补充。

然后我们看下它的整体流程。这里收集了每个玩家的位置。注意一下上面的 CentroidLocation，这个是所有玩家的中心位置。

可以看到，World Origin Rebasing 也在这里。不过我们等下再看原点偏移，先看看流送是怎么做的。

  1. 收集所有的玩家位置 Locations。
  2. 遍历每一个 Tile Level，看是否在某个玩家的对应 TileStreamingDistance 内。
  3. 加入可见的 Levels 集合或者不可见的 Levels 集合。
  4. CommitTileStreamingState 提交可见的和不可见的 Levels。

# 补充：

WorldComposition 大概配置方法，具体方法见官方文档

  1. 在 Level 的 World Settings 中勾选 Enable World Composition。
  2. 把需要流送的 Tile Levels 存放在 PersistentLevel 所在目录下 。
  3. 在 Levels 视图中打开 WorldComposition 视图可观察和配置 Tile Levels。
  4. 配置好流送的距离。
  5. 运行游戏则会自动根据玩家位置加载距离内的 Tiles。。

# 世界原点偏移：

  1. 拿到前面以所有玩家的中心 CentroidLocation。

  2. 看所有玩家的中心到原点的距离 是否超过了原点重置距离 RebaseOriginDistance。

  3. 设置新的目标原点位置。

  4. 下一帧 UWorld::Tick 时，如果原点位置发生了变化，依次调用各个系统的 ApplyWorldOffset。

  5. Scene->ApplyWorldOffset(Offset);

  6. PhysicsScene->ApplyWorldOffset(Offset);

  7. LevelToShift->ApplyWorldOffset(Offset, true);

  8. Actor->ApplyWorldOffset(Offset, bWorldShift);

  9. ActorComponent->ApplyWorldOffset(InOffset, bWorldShift);

  10. 其他系统的 ApplyOffset。
